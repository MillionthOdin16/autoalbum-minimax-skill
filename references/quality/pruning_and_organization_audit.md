# Pruning and Organization Audit

## Goal

Reduce context noise while preserving every file that improves track/album quality, MiniMax controllability, workflow enforcement, or agent learning.

## Removed as bloat

- Superseded versioned architecture maps from previous package iterations.
- Historical full `SKILL.md` archives.
- Generated critic-comment examples that were not part of active execution.
- Old file inventories and validation summaries.
- Duplicate source-index placeholders and stale version-specific copies.
- Python `__pycache__` artifacts.

## Kept intentionally

- All active delegated stage instructions.
- All schemas for project, track, packet, render, review, and release artifacts.
- Agent training needed for non-expert execution.
- MiniMax API/provider references.
- Style catalogs and fidelity rubrics.
- Controller script and machine-readable phase/gate files.

## Best-practice rationale

The skill now follows progressive disclosure: concise `SKILL.md`, active map, stage router, conditional deep references, and scripts for deterministic checks. This reduces context load while preserving depth for phases that require it.
