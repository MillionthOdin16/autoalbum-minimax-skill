# Track Style Card Template

Every track must have a style card before MiniMax packet compilation.

```json
{
  "track_number": 1,
  "title": "",
  "style_strictness": 4,
  "raw_user_style_request": "",
  "reference_anchors": [],
  "primary_style_profile": "",
  "secondary_style_profiles": [],
  "song_reference_profiles": [],
  "style_summary": "",
  "shared_album_dna": [],
  "track_style_delta": "",
  "tempo_feel": "",
  "groove": "",
  "drums": "",
  "bass_808": "",
  "harmony_melody": "",
  "instrument_palette": [],
  "vocal_delivery": "",
  "adlib_behavior": "",
  "hook_mechanics": "",
  "lyric_posture": "",
  "arrangement_architecture": "",
  "mix_space": "",
  "minimax_prompt_capsule": "",
  "hard_style_traits": [],
  "soft_style_traits": [],
  "avoid": [],
  "adjacent_non_confusion": [],
  "album_position_reason": "",
  "quality_risks": [],
  "style_card_score": 0
}
```

The style card is the bridge between catalog research and the MiniMax prompt. It must be specific enough that a different agent can compile a strong prompt without reading the whole catalog.
