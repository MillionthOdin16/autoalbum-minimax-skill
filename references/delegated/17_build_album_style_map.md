# Delegated: Build Album Style Map

## Task
Create `style_catalog_outputs/album_style_map.md`, the album-level style control document that keeps the project coherent while allowing track-to-track contrast.

## Read
- intent_intake.json
- `style_catalog_outputs/style_selection.json`
- `references/style_catalog/album_style_map_template.md`
- selected genre and artist profiles
- `foundation/album_thesis.md`
- `foundation/sound_signature.md` if already created
- `planning/tracklist.md` if already created

## Requirements
The album style map must include:

1. Raw user style request.
2. Reference anchors.
3. Primary and secondary lanes.
4. Shared sonic DNA.
5. Controlled variation plan.
6. Track-by-track style deltas.
7. Sequencing implications.
8. MiniMax compilation rules.
9. Do-not-list for generic or style-breaking traits.
10. Adjacent-track contrast logic.

## Output table

```markdown
| Track | Function | Style lane | Tempo feel | Groove | Vocal mode | Production delta | Why it belongs | Why distinct |
|---|---|---|---|---|---|---|---|---|
```

## Quality bar
Every track must have at least three shared DNA traits and at least two distinction traits. If a track cannot be distinguished, mark it for redesign before lyric drafting.
