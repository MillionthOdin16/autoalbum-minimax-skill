# AutoAlbum Operating Contract

## Purpose
AutoAlbum is a full-cycle album-control skill for MiniMax Music 2.6. Its job is not to merely write lyrics or prompts; its job is to maintain a professional album project from concept through final render selection.

## Required operating standard
Every stage must create an artifact that either:
1. improves the album concept,
2. improves songcraft,
3. improves style fidelity,
4. improves MiniMax controllability,
5. improves candidate selection,
6. improves sequencing, or
7. improves release/package readiness.

If a file or stage cannot prove one of those functions, remove it or merge it into a more useful stage.

## End-to-end responsibility
The skill owns:
- user intent capture and project configuration,
- style catalog resolution,
- album thesis and sonic DNA,
- tracklist/arc/energy design,
- lyrics and songcraft,
- style cards and reference fingerprints,
- MiniMax prompt and payload compilation,
- API parameter audits,
- variant strategy,
- render logging,
- post-generation review,
- regeneration decisions,
- final sequencing,
- release package.

MiniMax owns rendering, but AutoAlbum owns deciding whether a render is good enough.

## The professional bar
A track does not pass because it is listenable. It passes only if it has:
- a clear style lane,
- recognizable production/vocal/groove identity,
- a memorable hook or moment,
- lyric intent that survives singing,
- arrangement motion,
- no adjacent-track redundancy,
- a usable MiniMax packet,
- at least one reviewed render or explicit pending-render approval.

An album does not pass because the tracks share a genre. It passes only if the sequence feels authored: opener promise, early range, middle development, late payoff, and closer aftertaste.
