# Phase Gate Matrix

This matrix pairs controller-enforced structural checks with musical/creative quality gates.

| Phase | Structural controller gate | Musical/creative gate | Fail response |
|---|---|---|---|
| intake | Root control files exist | user intent classified, assumptions documented | repair config/intake |
| concept_option_search | concept tournament exists | selected concept is specific, fertile, non-generic | rerun tournament |
| style_resolution | style selection/map exists | concrete style controls, not vague labels | refine style resolution |
| style_voice_trials | trials exist | selected style/voice has strongest musical promise | run more trials |
| foundation | foundation docs exist | thesis, sound DNA, lyric/vocal identity cohere | revise foundation |
| canon_setup | canon/ledgers exist | rules prevent drift without killing creativity | repair canon/debts |
| planning_style_cards | tracklist/style cards exist | album flow, track differentiation, style deltas work | rebuild planning |
| motif_plant_payoff | motif ledger exists when applicable | motifs are intentional and transformed | revise motif plan |
| hook_title_tournament | tournament exists when required | hook/title is memorable and style-fit | rerun hook lab |
| arrangement_energy_design | energy review exists | form/tension/groove/arrangement support song | revise arrangement |
| lyrics_songcraft | lyrics + eval exist | lyrics are specific, performable, non-generic | rewrite lyrics |
| prosody_topline | prosody review exists | stress, breath, cadence, vowels, flow pass | revise scansion/topline |
| preproduction_brief | brief exists | groove/drums/bass/harmony/vocal/mix are actionable | refine brief |
| adversarial_song_edit | adversarial findings exist | no critical lyric/form/preproduction defects remain | patch/rewrite |
| packet_compilation | prompt/lyrics/packet exist | packet translates song into MiniMax controls | recompile packet |
| prompt_budget | budget report exists | prompt is concise, audible, provider-safe | compress prompt |
| adversarial_packet_edit | packet adversarial review exists | no critical prompt/API/style failure remains | patch packet |
| packet_evaluation | MiniMax/style/API evals exist | packet score and style fidelity pass | revise or reject packet |
| unified_quality_decision | unified eval exists | clear PASS/REVISE/REGENERATE/CUT decision | follow decision |
| variant_strategy | complete variant packets exist | variants explore meaningful search space | revise variants |
| payload_export | payload files exist | payloads reflect approved packets exactly | regenerate payloads |
| render_candidates | render logs/audio exist | render process is traceable | repair logging/download |
| post_render_review | comparison/selection exists | selected render wins on song, style, audio quality | regenerate/revise/cut |
| post_audio_rebuild | rebuilt plans exist | actual audio sequence works better than paper plan | resequence/rebuild |
| final_qc_release | release artifacts exist | final audio, metadata, sequence, transitions pass | repair release package |
| executive_producer_review | review exists | no major release blocker remains | revise/cut/resequence |
| completion_fit | completion reports exist | claim matches actual output goal | correct claim or artifacts |

## Controller vs quality evaluator

A controller gate can pass while the music is still bad. A quality gate can pass while the folder is incomplete. Both must pass when the phase is required.


## V23 additions

- `line_polish` is mandatory for vocal professional-quality tracks.
- `completion_fit` must include freshness, quote, style-catalog, tournament, prompt-lint, variant, payload, state-reconciliation, and independent-verifier checks.
